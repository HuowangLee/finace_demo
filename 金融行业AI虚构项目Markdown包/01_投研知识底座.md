# 投研知识底座

## 所属场景

投研分析

## 项目类型

投研人员工作台首页原型

## 甲方背景

**甲方：某股份制银行理财子公司投研部**

甲方管理多只固收、固收+、混合类理财产品，投研资料分散在券商研报、内部研究报告、债券募集说明书、评级报告、上市公司公告、宏观政策文件和会议纪要中。研究员需要反复查资料、比对口径、整理主体和行业信息，新员工也难以复用历史研究结论。

## 甲方需求

建设面向投研人员的 AI 知识底座，支持按主体、行业、指标、事件、风险事项进行检索、问答、摘要生成和来源溯源，提升资料检索效率和研究观点沉淀能力。

## AI+金融解决方案

1. 接入研报、公告、财报、评级报告、政策文件、内部纪要等多源资料。
2. 通过文档解析、版面识别、指标抽取和主题分类，形成统一的投研资料索引。
3. 基于 RAG、语义检索和权限控制，为研究员提供可溯源问答。
4. 围绕主体、行业、评级、财务指标、风险事件建立结构化标签体系。
5. 沉淀历史研究结论、风控意见和投资会议纪要，形成可复用的投研资产。

## 核心功能模块

| 模块 | 说明 |
|---|---|
| 研报解析 | 自动识别标题、行业、公司、指标、评级、风险提示。 |
| 主体检索 | 输入公司名、债券主体或行业关键词，返回相关资料。 |
| 投研问答 | 支持询问某主体的主要风险、财务变化和历史观点。 |
| 来源溯源 | 回答关联原始文档、页码、段落和版本。 |
| 观点沉淀 | 归档历史研究结论、调研纪要和风控意见。 |

## 页面首页摘要

面向股份制银行理财子公司投研部的日常工作首页。用户进入系统后，首先完成资料检索、继续处理未完成研究任务、查看待复核观点、跟踪资料更新，并从常用入口进入上传资料、投研问答和观点归档。

## 产品首页设计方案

### 设计定位

页面定位为研究员登录后的实际使用首页。首页需要回答研究员打开系统后的几个直接问题：今天要处理什么、从哪里检索资料、上次研究任务在哪里继续、哪些资料刚入库、哪些 AI 观点需要人工复核。

### 页面结构

1. 左侧为系统导航，包含首页、资料检索、投研问答、观点库、复核任务和各类资料入口。
2. 顶部为用户上下文和全局检索，支持选择资料范围后直接输入主体、行业、指标或事件。
3. 主区域展示“继续处理”，承接研究员上次未完成的主体跟踪、资料缺口和观点归档任务。
4. 常用入口提供上传资料、新建问答、写入观点库等高频操作。
5. 最近访问展示最近查看的主体、公告和内部纪要，帮助研究员快速回到工作现场。
6. 右侧展示待办、资料更新和知识库状态，突出复核、补资料、权限申请和解析失败等实际系统事项。

### 视觉与交互

- 使用企业后台产品风格，布局克制、信息密度适中，突出任务、检索、资料和复核状态。
- 页面内置独立 CSS 和 JavaScript，不依赖公共样式或公共渲染逻辑。
- 支持点击检索建议填充搜索框、点击导航切换状态、点击任务和入口给出操作反馈。
- 页面内容服务于实际使用路径，围绕投研知识底座一个产品首页展开。
